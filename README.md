# 语音情绪驱动虚拟头部

基于 Web Audio API 和 Three.js 的实时语音情绪识别与面部动画系统。全部代码本地运行，无外部API依赖。

## 快速启动

```bash
cd E:/coding_agent/emo
python -m http.server 8090
# 浏览器打开 http://localhost:8090
```

点击"开始录音"→ 允许麦克风权限 → 系统自动校准(约2秒) → 开始识别。

## 项目结构

```
emo/
├── index.html          # 主页面（含响应式CSS）
├── lib/three.min.js    # Three.js r128 本地副本
├── js/
│   ├── main.js         # 主入口，协调所有模块
│   ├── audio.js        # Web Audio API 特征提取
│   ├── emotion.js      # 阈值规则情绪识别 + 5帧多数滤波
│   ├── head.js         # Three.js 程序化3D头部（内置几何体+法线贴图）
│   ├── animation.js    # 情绪→表情映射 + 音量点头
│   └── ui.js           # UI：情绪标签/电平条/波形/频谱
└── README.md
```

## 功能特性

### 音频特征提取 (audio.js)
- **能量**：时域信号RMS²
- **响度**：RMS转dB
- **过零率(ZCR)**：时域过零频率
- **频谱质心**：加权频率中心
- **音高(F0)**：自相关法，50-500Hz
- **MFCC**：20个Mel滤波器→DCT取前3阶

### 情绪识别规则 (emotion.js)
| 情绪 | 能量 | ZCR | 频谱质心 | 响度 |
|------|------|-----|----------|------|
| 高兴 | 中高 | 中等 | 偏高 | 中高 |
| 悲伤 | 低 | 低 | 低 | 低 |
| 生气 | 高 | 高 | 高 | 高 |
| 中性 | 基线 | 基线 | 基线 | 基线 |

**防闪烁机制**：
1. EMA平滑原始特征 (α=0.35)
2. 5帧投票窗口
3. 滞后阈值：新情绪需在5帧中获>=3票才切换

**自适应校准**：启动后前50帧建立个人基线均值/标准差。

### 面部动画 (head.js + animation.js)
| 情绪 | 嘴巴 | 眼睛 | 眉毛 |
|------|------|------|------|
| 高兴 | 嘴角上扬、微张 | 眯眼 | 略抬 |
| 悲伤 | 嘴角下拉 | 正常 | "八"字 |
| 生气 | 微张 | 瞪眼 | 皱紧 |
| 中性 | 放松 | 正常 | 正常 |

- **音量→点头**：短时能量驱动头部微幅晃动
- **说话→嘴部**：响度>-35dB时嘴部跟随张合
- **呼吸微动**：正弦波模拟自然呼吸
- **眨眼**：2-5秒间隔自动眨眼

### 3D模型 (head.js)
纯Three.js内置几何体构建，无外部GLTF依赖：
- SphereGeometry变形头骨 + DataTexture程序化法线贴图
- ExtrudeGeometry眼睑/眉毛/嘴唇
- MeshStandardMaterial + HemisphereLight真实光照

### 调试
打开浏览器F12控制台可看到：
- `[主] 麦克风已启动` - 启动确认
- `[情绪] 基线校准完成` - 校准结果
- `[特征] {...}` - 每30帧输出一次当前特征和情绪
